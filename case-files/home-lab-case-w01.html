SSH Brute Force & Account Lockout

CASE-W01  ·  Jun 11, 2026  ·  Home Lab — Wazuh + Sysmon
Severidad: High SSH / Port 22 Brute Force Resuelto Windows 10 Home Wazuh 4.7.5 Sysmon v15.20

Resumen del Incidente

Se detectó un ataque de fuerza bruta contra el servicio SSH (puerto 22) del endpoint DESKTOP-CC2O1S4 (192.168.0.103). El atacante, desde 192.168.0.108 (Kali Linux), lanzó múltiples intentos de autenticación en un lapso de aproximadamente 11 segundos usando la herramienta Hydra con un diccionario de contraseñas. Windows activó la política de bloqueo de cuenta (Event 4740) tras superar el umbral de fallos. Wazuh capturó la secuencia completa: fallos de autenticación (4625), bloqueo de cuenta (4740) y logins exitosos posteriores (4624 Logon Type 3).

La correlación de la IP fuente del atacante no fue posible directamente desde los eventos 4625 del Security Log, ya que OpenSSH en Windows no popula el campo Dirección de red de origen. La atribución fue confirmada mediante el canal OpenSSH/Operational en el Event Viewer de la máquina víctima.

Acciones Tomadas

Detección: Wazuh generó alertas nivel 5 por múltiples 4625 consecutivos (regla 60122) y alerta nivel 9 por bloqueo de cuenta (regla 60115 — Event 4740).
Análisis: Se correlacionaron los eventos 4625, 4740 y 4624 por timestamp para reconstruir la secuencia del ataque. La IP fuente fue confirmada mediante el canal OpenSSH/Operational del Event Viewer en la máquina víctima, ya que el Security Log de Windows no popula el campo Dirección de red de origen en eventos SSH.

Contención simulada: En un entorno de producción se recomendarían las siguientes acciones inmediatas:

# Bloquear IP atacante via Windows Firewall New-NetFirewallRule -DisplayName "Block Attacker" -Direction Inbound ` -RemoteAddress 192.168.0.108 -Action Block # Desbloquear cuenta tras investigación net user jose /active:yes # Revisar intentos desde OpenSSH Operational log Get-WinEvent -LogName "OpenSSH/Operational" | ` Where-Object {$_.Message -like "*192.168.0.108*"} | ` Format-List TimeCreated, Message

Limitaciones Técnicas Identificadas

OpenSSH en Windows no registra IP fuente en Security Log: Los eventos 4625 generados por OpenSSH no populan el campo Dirección de red de origen, a diferencia de lo que ocurre con autenticación SMB o RDP. La correlación de IP atacante requiere fuentes adicionales: canal OpenSSH/Operational, logs de firewall perimetral, o NetFlow. Este es el comportamiento real del sistema, no una limitación del lab.

Lecciones Aprendidas

01 Un volumen alto de Event 4625 en un rango de segundos es el indicador primario de brute force, independientemente de si la IP fuente es visible en el log.
02 El Event 4740 (account lockout) es una señal de alta confianza — confirma que el volumen de fallos superó el umbral configurado, lo que en producción requiere respuesta inmediata.
03 La atribución completa en un SIEM requiere correlación de múltiples fuentes. Depender exclusivamente del Security Log de Windows para obtener la IP atacante es insuficiente cuando el vector es SSH.
04 Windows Firewall bloqueaba ICMP por defecto, lo que inicialmente impedía la conectividad entre Kali y el endpoint. En un entorno real, un atacante no dependería de ICMP para lanzar el ataque — usaría directamente el puerto 22.
