// CASE-002 · SOC153 True Positive Cerrado Medium

PowerShell Malicioso —
KOI-STEALER via Script Descargado

Ejecución de script PowerShell malicioso en endpoint Windows 10 tras descarga desde URL externa. El análisis confirmó conexión activa con infraestructura C2 asociada al infostealer KOI-STEALER, capaz de robar credenciales, cookies y datos del navegador.

// fecha 14 Mar 2024 · 05:23 PM
// severidad Media
// fuente LetsDefend · SOC153
// clasificación True Positive
// malware KOI-STEALER

// 01_resumen

Resumen ejecutivo

El EDR detectó la ejecución de payload_1.ps1 en el endpoint Tony (172.16.17.206), un script PowerShell descargado desde una URL externa. El análisis del hash en VirusTotal confirmó detección maliciosa por 34 de 62 motores, y el análisis de logs de red reveló comunicación activa con el dominio kionagranada.com (161.22.46.148), infraestructura C2 asociada al infostealer KOI-STEALER.

El script fue ejecutado con bypass de política de ejecución, técnica común para evadir controles de seguridad en Windows. El endpoint fue aislado de la red para contener posible exfiltración de credenciales y datos del navegador.

// veredicto

True Positive confirmado. Ejecución de infostealer KOI-STEALER con comunicación C2 activa. Endpoint Tony aislado. Se recomienda análisis forense completo del sistema, reseteo de credenciales del usuario afectado y revisión de otros endpoints en el mismo segmento de red.

// 03_comando

Comando malicioso ejecutado

El log del endpoint reveló la siguiente línea de ejecución, confirmando que el script fue ejecutado con bypass deliberado de la política de ejecución de PowerShell:

# Proceso padre: PowerShell.exe
"C:\Windows\System32\WindowsPowerShell\v1.0\powershell.exe"
"-Command"
"if((Get-ExecutionPolicy) -ne 'AllSigned') {
    Set-ExecutionPolicy -Scope Process Bypass
};
& 'C:\Users\LetsDefend\Downloads\payload_1.ps1\payload_1.ps1'"

El uso de Set-ExecutionPolicy -Scope Process Bypass es una técnica de evasión clásica (MITRE T1059.001) que permite ejecutar scripts sin restricciones en el proceso actual, evitando los controles de política configurados en el sistema.

// 07_acciones

Acciones tomadas

01

Verificación del hash: SHA256 del archivo consultado en VirusTotal — 34/62 motores lo detectan como malicioso. La threat intel interna no tenía registro previo.

02

Análisis del comando ejecutado: Identificado uso de ExecutionPolicy Bypass en PowerShell, técnica de evasión deliberada (T1059.001). Confirmada ejecución del script descargado.

03

Investigación de tráfico de red: Revisión de logs Sysmon EventID 22 — DNS Query reveló resolución de kionagranada.com por parte de PowerShell, confirmando comunicación C2 activa.

04

Verificación de propagación: Revisión del segmento de red — solo el endpoint Tony (172.16.17.206) presentó actividad relacionada. No se detectó movimiento lateral.

05

Aislamiento del endpoint: Tony aislado de la red para contener posible exfiltración en curso de credenciales y datos del navegador por KOI-STEALER.

06

Clasificación y escalamiento: True Positive confirmado. Caso escalado para análisis forense completo y reseteo de credenciales del usuario afectado.

Nota de análisis: La conexión C2 fue inicialmente no reportada al no estar explícita en el comando de ejecución. Fue identificada posteriormente mediante correlación con logs Sysmon (EventID 22), confirmando que el análisis de red es indispensable para casos de malware — el comando por sí solo no revela toda la actividad.

// 08_recomendaciones

Recomendaciones

  • Resetear todas las credenciales del usuario Tony — KOI-STEALER tiene capacidad de robar contraseñas del navegador y gestores de contraseñas
  • Realizar análisis forense completo del endpoint para determinar qué datos fueron exfiltrados durante el tiempo activo del malware
  • Bloquear en firewall las IPs y dominios IOC: kionagranada.com y 161.22.46.148
  • Implementar política de restricción de PowerShell (Constrained Language Mode) para usuarios no administrativos
  • Configurar AppLocker o WDAC para bloquear ejecución de scripts desde la carpeta Downloads
  • Revisar si otros endpoints descargaron archivos desde la misma URL de S3
  • Habilitar Script Block Logging en PowerShell para capturar contenido de scripts en el futuro

// 09_lecciones

Lecciones aprendidas

El comando no cuenta toda la historia. La ejecución del script no revelaba directamente la conexión C2. Solo el análisis de logs de red (Sysmon EventID 22) confirmó la comunicación con kionagranada.com. El análisis de red es obligatorio en casos de malware.

ExecutionPolicy Bypass es una señal de alerta inmediata. Cualquier proceso que use Set-ExecutionPolicy -Scope Process Bypass debe tratarse como sospechoso por defecto — es una técnica de evasión deliberada.

La threat intel interna tiene límites. El hash no aparecía en la base interna de LetsDefend pero sí en VirusTotal con 34 detecciones. Siempre consultar fuentes externas cuando la intel interna no arroja resultados.

KOI-STEALER exige respuesta urgente. Un infostealer activo con C2 establecido significa que las credenciales pueden estar siendo exfiltradas en tiempo real. El aislamiento inmediato no es opcional.
